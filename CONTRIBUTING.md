# Contributing to Rwanda Mineral Data Interoperability Standard

Thank you for your interest in contributing to the Rwanda Mineral Data Interoperability Standard Version 2.3.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:
1. Check if the issue already exists
2. Create a new issue with:
   - Clear description
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior
   - Relevant examples

### Proposing Changes

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test your changes
5. Submit a pull request

### Code Standards

- Follow existing code style
- **Use camelCase for all field names** (as per semantic model)
- **Use integer codes for status values** (0, 1, 2, 3 for certification; 0, 1, 2 for activity)
- **Validate ICGLR ID format**: `CC-[Lat]-[Long]-NNNNN`
- Include comments for complex logic
- Update documentation as needed
- Add tests for new features

### Documentation

- Keep documentation up to date
- Use clear, concise language
- Include examples where helpful
- Update README if structure changes
- Reference semantic model entities (MD.01, MD.02, etc.)
- Document code lists (MDC.01 - MDC.06)

## Review Process

All contributions will be reviewed by Rwanda Mines, Petroleum and Gas Board (RMB). The review process may take time as we ensure quality and alignment with the Rwanda Mineral Data Interoperability Standard.

## Contact

For questions about contributing, contact the Rwanda Mines, Petroleum and Gas Board (RMB).
